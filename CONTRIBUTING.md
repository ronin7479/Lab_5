# Contributing

Дякуємо за інтерес до внеску в цей проєкт! Будь ласка, дотримуйтесь таких правил:

## 1. Створюйте issue перед великими змінами
Якщо плануєте нову функцію або суттєву зміну — спочатку створіть issue, щоб обговорити ідею.

## 2. Використовуйте окремі гілки для роботи
Створюйте гілку з назвою у форматі:
- `feat/...` — для нових функцій
- `fix/...` — для виправлення помилок
- `docs/...` — для документації
- `chore/...` — для технічних змін

Приклад:
git checkout -b feat/add-user-auth

shell
Копировать код

## 3. Формат комітів
Дотримуйтесь стандарту:
type(scope): короткий опис

makefile
Копировать код

Приклади:
feat(auth): add login functionality
fix(db): correct connection string
docs(readme): update project description

markdown
Копировать код

## 4. Створюйте Pull Request
Після завершення роботи:
1. Запуште гілку
2. Створіть Pull Request (Merge Request)
3. Опишіть зміни та додайте посилання на issue, якщо воно є

## 5. Дотримуйтеся стилю коду
- Використовуйте однакове форматування
- Дотримуйтесь стилю проєкту (лінтер/форматер, якщо вказано)
